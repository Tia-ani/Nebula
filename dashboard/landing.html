01 — SUBMIT
Developer sends tasks
Array of prompts via API. Each becomes a separate chunk.
02 — ENCRYPT
AES-256 per session
Master creates a unique key per worker. Destroyed on disconnect.
03 — DISTRIBUTE
Chunks fan out
Idle devices process in parallel. Worker dies → auto-reassign.
04 — RETURN
Results assembled
Back to developer in order. Contributor earns credits per task.
